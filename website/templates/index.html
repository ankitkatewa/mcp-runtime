Spec 00 · Kubernetes MCP Runtime · Alpha

Operate MCP servers like infrastructure, not side projects.

MCP Runtime gives platform teams a Kubernetes-native control plane for deploying, governing, and observing internal Model Context Protocol services. CRDs define the service. Sentinel governs the live request path. Operators keep the cluster honest.

01 / Definition

One manifest becomes the governed route.

mcpserver.yaml mcpruntime.org/v1alpha1 
apiVersion: mcpruntime.org/v1alpha1
kind: MCPServer
metadata:
  name: payments
spec:
  image: registry.example.com/payments-mcp:v1.0.0
  port: 8088
  publicPathPrefix: payments
  gateway:
    enabled: true
  analytics:
    enabled: true

Apply once. The runtime owns rollout, routing, policy attachment, and telemetry from there.

  • Operator reconciles Kubernetes primitives from the CRD.
  • Gateway keeps access decisions on the request path.
  • Sentinel records policy outcomes and service analytics.
02 / Architecture

The platform is split across three operating planes.

A

Runtime

Cluster bootstrap, registry, ingress, operator reconciliation, and rollout for every MCP server.

B

Access

Grants and sessions as first-class CRDs with explicit consent, trust ceilings, expiry, and revocation.

C

Sentinel

Policy, audit, and observability on the live request path through a bundled gateway and dashboards.

03 / Capabilities

Built for platform ownership.

The product surface is intentionally operational: install the control plane, describe servers, issue access, then watch policy and traffic from one place.

Declarative deployments

Define MCP servers once as Kubernetes CRDs. The operator handles services, rollouts, routes, and updates.

Access without ambiguity

Grants and sessions make consent, trust level, expiry, and revocation inspectable Kubernetes resources.

Request-path analytics

Observe every MCP call with policy decisions, timing, tool names, status, and dashboard-ready event data.

Immediate policy enforcement

Sentinel evaluates live traffic at the gateway, so governance is applied before tools execute.

Audit-ready traces

Every governed request is recorded for security review, incident response, and compliance workflows.

Bundled dashboards

Grafana, Prometheus, API, ingest, processor, and UI services ship as part of the local evaluation stack.

04 / Runbook

From empty cluster to governed MCP endpoint.

  1. Build go build -o bin/mcp-runtime ./cmd/mcp-runtime
  2. Bootstrap ./bin/mcp-runtime bootstrap
  3. Install ./bin/mcp-runtime setup --test-mode
  4. Verify ./bin/mcp-runtime status
05 / Use Cases

Where the runtime earns its keep.

AI tool use

Safely expose internal tools to agents with controlled access, request inspection, and audit trails.

Internal APIs

Govern access to databases, services, and data APIs without modifying every application.

Cross-team integrations

Grant temporary, reviewable access between teams with clear trust ceilings and easy revocation.

Compliance review

Give security and compliance teams searchable request history instead of scattered service logs.
